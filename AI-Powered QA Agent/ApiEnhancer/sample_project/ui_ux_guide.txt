================================================================================
FORM VALIDATION UI/UX
================================================================================

Error Message Display Rules:
1. Form validation errors MUST be displayed in RED text
2. Error messages appear BELOW the respective input field
3. Error messages have 5px margin-top spacing
4. Error font size: 14px
5. Errors are hidden by default (display: none)
6. Errors become visible when validation fails
7. Errors should disappear when the field is corrected

Required Field Indicators:
- All required fields marked with asterisk (*) after label
- Required fields: Full Name, Email Address, Shipping Address
- Shipping Method and Payment Method are also required (radio buttons)

Validation Behavior:
- Validation triggers on form submission (Pay Now button click)
- Multiple errors can be displayed simultaneously
- Each field has its own error message element
- Form does not submit if validation fails

Specific Error Messages:
- Name field: "Name is required"
- Email field: "Please enter a valid email address"
- Address field: "Address is required"
- Empty cart: "Your cart is empty. Please add items before checkout." (alert)

================================================================================
ACCESSIBILITY & USABILITY
================================================================================

Form Labels:
- All form inputs have associated labels
- Labels use 'for' attribute matching input ID
- Labels displayed above inputs
- Clear, descriptive label text

Input Fields:
- All inputs have placeholder text
- Placeholder provides example or instruction
- Inputs are clearly bordered and visible
- Adequate padding for easy clicking/tapping

Radio Buttons:
- Grouped by name attribute
- Clear labels for each option
- Default selections specified (Standard shipping, Credit Card)
- Easy to click/tap

Responsive Design:
- Product grid adapts to screen size
- Minimum 250px card width maintains readability
- Layout adjusts for mobile and desktop
- Full-width Pay Now button works on all screen sizes

Visual Feedback:
- Hover effects on interactive elements
- Color changes on button hover
- Card lift effect on product hover
- Clear visual distinction between states
